============================


Enemy : 소유주

StateMachine : 상태 판별해서 상태 바꿔줌

State : 상태들 찍어낼 부모(원본)스크립트

State_Idle : State 형식인, '상태' 구현한 스크립트


===========================

FSM : 이전상태가 다음 상태를 지정함
상태 개수 만큼만 각각 상태별 스크립트가 존재

유니티에서의 기본 애니메이터 - 애니메이션 클립 상관관계 세팅과 동일




HFSM : 이전상태가 다음 부모상태를 지정함
상태 개수 + 부모상태 개수 / 혹은 부모상태 개수만큼 스크립트 존재. 
부모상태에서 다음 세부 상태 지정해줌

유니티에서의 애니메이터 - create subStateMachine 하여 처리하는 것과 비슷함.




BT : 상태 판별기 존재. 상태 판별기가 다음 상태를 지정해줌. 다음 상태는 부모상태일 수도 있고, 그 상태 자체일 수도 있고 취향. 
정식 BT는 상태 판별기가 각 상태가 가능한지 책임연쇄같은걸로 체크 해서 지정해줌. + 그래프 처럼 각 상태에 대한 가중치 존재하여, 가중치로 계산하여 진행함.
